## Условие

В бэкруме аутдор-хаба, где по ночам собирают рюкзаки к первым электричкам за город, на стене до сих пор висит старый терминал учёта провизии. Через него когда-то списывали батончики, дрипы, изотоник и пакеты с сублиматами. Терминал собран из складской панели, зелёных светодиодов и очень плохих инженерных решений: снаружи — тёмный пластик и толстенный экран, внутри — эксель.

Разберитесь, как получить доступ к неограниченным батончикам.

Терминал учёта провизии: docs.google.com/spreadsheets/d/1LDAeCB7-vl7IhpZFvCsZCB1Sdn59m5wsccmAsTOftcA/edit

## Наблюдения

- Книга — Google Sheets с 1 видимым листом `Panel` и 3 скрытыми: `s1`, `s2`, `s3`.
- Юзер вводит 8 строчных букв (a-z) в ячейки `Panel!Z6, AD6, AH6, AL6, AP6, AT6, AX6, BB6`.
- Баннер `Panel!B11` показывает «ДОСТУП РАЗРЕШЕН» только если все 8 символов из [a-z] **и** прошли хеш-проверку на `s1!J14:Q14`.
- Литерала `alfa{block_2_pixel_5_l3ast_5ignif1cant_bi7}` в книге нет ни в каком виде — grep по xlsx возвращает 0 совпадений. Флаг **отрисовывается** как растровое изображение 80×26 на листе `Panel` условным форматированием по битам из `s3!P2:Y27`.
- В `s1!A23:B48` — lookup-таблица a→97, b→98, ..., z→122. Поэтому пароль гарантированно состоит из строчных ASCII-символов.

## Решение

### Этап 1. Реверс валидации пароля (s1)

Все вычисления ведутся mod 251. Входные данные: вектор `x[0..7]` — ASCII-коды символов пароля, ограниченные [97, 122].

Три раунда «свёртки»:

```
y8[k]  = (M1 · x + e1)[k]  mod 251
         M1 = s1!O30:V37,  e1 = s1!E30:L30

y10[k] = (M2a · y8 + M2b · x + e2)[k]  mod 251
         M2a = s1!X30:AE37,  M2b = s1!AG30:AN37,  e2 = s1!E31:L31

y12[k] = (M3a · y10 + M3b · y8 + b6 · v34 + c6 · v35 + d6 · v36 + e3)[k]  mod 251
         M3a = s1!AP30:AW37,  M3b = s1!AY30:BF37
         v34 = s1!E34:L34,  v35 = s1!E35:L35,  v36 = s1!E36:L36
         b6 = (m_b6 · x) mod 251  (m_b6 = s1!E40:L40, аналогично c6, d6)
         e3 = s1!E32:L32
```

Целевой вектор: `s1!A18:H18 = [92, 97, 27, 240, 199, 80, 217, 23]`.

Ключевое наблюдение: `b6`, `c6`, `d6` — скалярные значения, линейные в `x`. Умножение «(линейное в x) × (константный вектор)» остаётся линейным. Значит вся система `y12 = M_total · x + c_total mod 251` — обычная линейная система 8×8 над полем F₂₅₁.

Решается Gaussian elimination за миллисекунды. Пароль: **`xlmatrix`**.

Примечание по тупикам: первая попытка через Z3 (`solve/solve.py`) работает, но зависает на ~100% CPU из-за кодирования modular arithmetic в неограниченных целых. Линейная алгебра на порядок быстрее и надёжнее.

### Этап 2. Реверс рендеринга флага (s2 → s3)

`s2!B4:E4` упаковывает 8 байт пароля в 4×16-битных слова: `B4 = ord(x[0]) + 256·ord(x[1])` и т.д.

`s2!{F,G,B,C,D,E}{r}` для `r ∈ [6, 70]` — детерминированный PRG, 65 итераций, на каждой 4 байта выходного потока:

```
F  = (((B<<5) ^ (C>>3)) + (D ^ (E<<1)) + 25713 + n*911)  mod 65536
G  = (((C<<4) ^ (D>>2)) + (E ^ (B<<2)) + 12345 + n*1499) mod 65536
B' = (C ^ F)  mod 65536
C' = (D ^ G)  mod 65536
D' = (E ^ (B' + 54321 + n*577))  mod 65536
E' = (B ^ (C' + 22222 + n*313))  mod 65536
out += [B'&0xFF, (C'>>8)&0xFF, D'&0xFF, (E'>>8)&0xFF]
```

где `n` — номер итерации (A6=0 ... A70=64).

65×4 = 260 байт стрима → `s3!B2:B261`.

`s3!P2:Y27` = `BITXOR(s3!D2:M27, s3!B2:B261)` — 26×10 байт plaintext.

### Этап 3. Рендер 26×80 битмапы

Каждый из 260 plaintext-байт кодирует 8 пикселей строки (MSB-first). Условное форматирование листа `Panel` рисует зелёный=бит 1 / чёрный=бит 0 на сетке 80×26.

Скрипт `solve/render_png.py` симулирует PRG `s2`, XOR'ит с константным паддингом из xlsx, рисует MSB и LSB варианты в PGM, конвертит в PNG.

Результат (`solve/flag_msb.png`):

- верхняя строка: `alfa{here_is`
- нижняя строка: `_your_snack}`
