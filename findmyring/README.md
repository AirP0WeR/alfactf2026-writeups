## Условие

Недавно вы купили фитнес-кольцо нового поколения: держит зарядку неделю, не жужжит уведомлениями, ещё и пульс снимает во время бега на дорожке. Вы надели его на палец и не снимали ни на пробежке, ни на скалодроме, ни в душе.

А потом случился BASScamp — трёхдневный всемирный рэйв, где танцполы раскиданы по всему свету от Тбилиси до Ушуаи. Во время то ли сета на рассвете, то ли стихийной йоги на закате, кольцо слетело с пальца. Теперь оно лежит в какой-то точке земного шара — может, в песке на пляже Бали, может, в траве у исландского водопада, может, под диджейским пультом в Боготе.

К счастью, знакомый радиоэлектроник привёз вам классный подгон: блютус-антенну, которая поможет найти кольцо издалека. Можно включить её в любом месте, выкрутить радиус сканирования, и она запищит, если примет BLE-маячок от кольца.

## Решение

Категория — **Coding / Algorithm Optimization**. Задача геолокации с ограниченным количеством запросов к API.

### API анализ

Сервер предоставляет 3 эндпоинта:

- `POST /api/start` — начинает новую игру, возвращает `attempts_left: 46`
- `POST /api/antenna` — проверяет область `(lat, lon, radius_m)`, возвращает `found: bool` 
- `POST /api/guess` — финальная попытка угадать точное местоположение

Цель: найти случайную точку на Земле за ≤45 проб антенны + 1 финальная попытка.

### Алгоритм: Information-optimal Monte Carlo Binary Search

Разработан многоуровневый алгоритм пространственного поиска на сфере:

1. **Инициализация**: генерируется 1M случайных точек равномерно на сфере
2. **Итеративное зондирование**: 
   - Выбирается оптимальный центр пробы с помощью greedy-поиска
   - Радиус = медианное расстояние до точек в кластере  
   - После каждого зондирования отбрасываются точки, не удовлетворяющие ограничению
3. **Ресэмплинг**: когда точек становится <20K, выполняется bootstrap+jitter для сохранения плотности
4. **Критерий останова**: диаметр кластера <2м или исчерпаны попытки

### Ключевые оптимизации

- **Сферическая геометрия**: использование формулы гаверсинуса для расчёта расстояний на Earth (R=6,371 км)
- **Cap-uniform sampling**: генерация точек в сферическом сегменте с правильным распределением
- **Centroid via unit vector averaging**: устойчивый к антиподальным точкам
- **Bootstrap+jitter resampling**: сохраняет пространственное распределение при увеличении плотности
